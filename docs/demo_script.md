# 15-Minute Presentation Guide

1. Problem and user story, 1 minute: deadline and meeting prep for students and knowledge workers.
2. Architecture, 3 minutes: orchestrator, planner, retriever, risk/safety, response agent, MCP-like tools.
3. Safety demo, 2 minutes: run an unsafe cancellation request and show the blocked action.
4. Normal demo, 2 minutes: run a meeting prep query and explain evidence, checklist, risks, and safe next actions. If using LLM mode, run with `--llm` after exporting `OPENAI_API_KEY`.
5. Eval harness, 3 minutes: show golden cases, metrics, judge fallback, and result files.
6. Experiment analysis, 2 minutes: compare baseline and optimized modes.
7. Failures and tradeoff, 2 minutes: discuss remaining weaknesses and deterministic mock-data tradeoff.
